# Комп'ютерні мережі та мережева безпека

## Модель OSI

Семирівнева еталонна модель мережевої взаємодії:

1. **Фізичний (L1)** — кабелі, сигнали, роз'єми
2. **Канальний (L2)** — MAC-адреси, Ethernet, VLAN, комутатори
3. **Мережевий (L3)** — IP-адресація, маршрутизація, ICMP
4. **Транспортний (L4)** — TCP (надійний), UDP (ненадійний), порти
5. **Сеансовий (L5)** — управління сеансами зв'язку
6. **Представлення (L6)** — кодування, стиснення, шифрування даних
7. **Прикладний (L7)** — HTTP, DNS, SMTP, FTP, SSH

## Модель TCP/IP

Чотирирівнева модель, що використовується в Інтернеті:

1. **Канальний** — Ethernet, Wi-Fi, ARP
2. **Мережевий** — IP, ICMP
3. **Транспортний** — TCP, UDP
4. **Прикладний** — HTTP, HTTPS, DNS, SSH, TLS

## Адресація

### IP-адресація

- **IPv4** — 32 біти (4 октети), наприклад 192.168.1.1
- **IPv6** — 128 біт, наприклад 2001:0db8::1
- **Маска підмережі** — визначає межі мережі (наприклад /24 = 255.255.255.0)
- **NAT** — трансляція мережевих адрес (приватні ↔ публічні)

### MAC-адресація

- 48 біт (6 байт), наприклад AA:BB:CC:DD:EE:FF
- Унікальний ідентифікатор мережевого інтерфейсу
- Працює на канальному рівні (L2)

### Розрахунки підмереж (CIDR)

CIDR (Classless Inter-Domain Routing) — безкласова адресація, що дозволяє гнучко розділяти IP-простір.

**Маска підмережі** визначає, яка частина IP-адреси належить мережі, а яка — хосту:

- **/24** = 255.255.255.0 → 256 адрес, 254 хости
- **/25** = 255.255.255.128 → 128 адрес, 126 хостів
- **/26** = 255.255.255.192 → 64 адреси, 62 хости
- **/27** = 255.255.255.224 → 32 адреси, 30 хостів
- **/28** = 255.255.255.240 → 16 адрес, 14 хостів
- **/30** = 255.255.255.252 → 4 адреси, 2 хости (point-to-point з'єднання)

**Формула:** кількість хостів = 2^(32 - prefix) - 2 (мінус адреса мережі та широкомовна адреса).

**Приклад розрахунку:** визначити мережу для хоста 168.14.17.254/26.

- Маска /26 означає блоки по 64 адреси (2^(32-26) = 64)
- Останній октет 254 ÷ 64 = 3 (ціла частина), 3 × 64 = 192
- Мережа: **168.14.17.192/26** (діапазон 192–255)
- Перша доступна адреса хоста: 168.14.17.193
- Остання доступна адреса хоста: 168.14.17.254
- Широкомовна адреса: 168.14.17.255

## Мережеве обладнання

- **Комутатор (switch)** — працює на L2, пересилає кадри за MAC-адресою
- **Маршрутизатор (router)** — працює на L3, пересилає пакети між мережами за IP
- **Точка доступу (AP)** — забезпечує бездротовий доступ (Wi-Fi)

## Протоколи транспортного рівня

### TCP детально

TCP (Transmission Control Protocol) — надійний протокол із встановленням з'єднання.

**Встановлення з'єднання — three-way handshake:**

1. **SYN** — клієнт надсилає сегмент з прапорцем SYN та початковим номером послідовності (ISN)
2. **SYN-ACK** — сервер відповідає сегментом з прапорцями SYN та ACK, вказуючи свій ISN
3. **ACK** — клієнт підтверджує отримання, з'єднання встановлено

**Прапорці TCP:**

- **SYN** — ініціювання з'єднання, синхронізація номерів послідовності
- **ACK** — підтвердження отримання даних
- **FIN** — завершення з'єднання (немає більше даних для відправки)
- **RST** — негайний розрив з'єднання (помилка або відмова)
- **PSH** — негайна доставка даних додатку без буферизації
- **URG** — термінові дані, що мають пріоритет обробки

**Підтвердження доставки та повторна передача:**

- Кожен відправлений байт має номер послідовності (sequence number)
- Отримувач відповідає ACK з номером наступного очікуваного байта
- Якщо ACK не отримано протягом тайм-ауту (RTO — Retransmission Timeout), відправник повторює передачу
- Механізм Fast Retransmit — повторна передача після трьох дублікатів ACK

**Розмір вікна (Window Size):**

- Визначає обсяг даних, які можна відправити без підтвердження
- Механізм управління потоком (flow control) — отримувач регулює вікно залежно від свого буфера
- Sliding Window — вікно "ковзає" вперед у міру отримання підтверджень

**Congestion Control (управління перевантаженням):**

- **Slow Start** — експоненційне збільшення вікна від 1 MSS до порогу (ssthresh)
- **Congestion Avoidance** — лінійне збільшення вікна після досягнення порогу
- **Fast Recovery** — швидке відновлення після втрати пакета без повернення до Slow Start

**Завершення з'єднання — four-way termination:**

1. **FIN** — сторона A повідомляє про завершення передачі
2. **ACK** — сторона B підтверджує отримання FIN
3. **FIN** — сторона B також завершує передачу
4. **ACK** — сторона A підтверджує, з'єднання закрито

### UDP

UDP (User Datagram Protocol) — простий протокол без встановлення з'єднання.

**Характеристики:**

- Без встановлення з'єднання (connectionless) — дані відправляються без попереднього handshake
- Без підтвердження доставки — відправник не знає, чи дані дійшли
- Без гарантії порядку — пакети можуть прийти в довільному порядку
- Мінімальний заголовок (8 байт проти 20+ у TCP) — менші накладні витрати
- Менша затримка (latency) — немає витрат на встановлення з'єднання

**Використання UDP:**

- **DNS** — швидкі запити на розв'язання імен (порт 53)
- **DHCP** — автоматичне призначення IP-адрес (порти 67/68)
- **SNMP** — моніторинг мережевого обладнання (порти 161/162)
- **VoIP** — голосовий зв'язок через інтернет (SIP, RTP)
- **Стрімінг відео/аудіо** — де втрата окремих кадрів допустима
- **Онлайн-ігри** — де швидкість важливіша за надійність

## Протоколи канального та мережевого рівнів

### ARP

ARP (Address Resolution Protocol) — протокол визначення MAC-адреси за відомою IP-адресою.

**Принцип роботи:**

1. Хост знає IP-адресу призначення, але не знає MAC-адресу
2. Відправляє ARP-запит (broadcast) на всю локальну мережу: "Хто має IP x.x.x.x?"
3. Хост з відповідною IP-адресою відповідає ARP-відповіддю (unicast) зі своєю MAC-адресою
4. Відправник зберігає відповідність IP → MAC в ARP-таблиці (кеші)

**ARP-таблиця:**

- Зберігає пари IP → MAC для пришвидшення подальших комунікацій
- Записи мають обмежений час життя (TTL), після чого видаляються
- Переглянути таблицю можна командою `arp -a`

**Gratuitous ARP:**

- ARP-запит, де відправник запитує свою власну IP-адресу
- Використовується для оголошення своєї присутності або оновлення ARP-кешу інших хостів
- Застосовується при зміні MAC-адреси або для виявлення конфліктів IP-адрес

**ARP-spoofing (ARP-poisoning):**

- Атакуючий надсилає підроблені ARP-відповіді у мережу
- Жертва зберігає хибну відповідність IP → MAC атакуючого у своєму кеші
- Весь трафік жертви починає проходити через атакуючого (Man-in-the-Middle)
- Захист: Dynamic ARP Inspection (DAI), статичні ARP-записи, засоби виявлення (arpwatch)

### ICMP

ICMP (Internet Control Message Protocol) — протокол для передачі діагностичних та службових повідомлень.

**Основні типи повідомлень:**

- **Type 8 / Type 0 — Echo Request / Echo Reply** — основа утиліти `ping`, перевірка доступності хоста
- **Type 11 — Time Exceeded** — час життя пакета (TTL) вичерпано; основа утиліти `traceroute`
- **Type 3 — Destination Unreachable** — хост або порт недоступний; різні коди причин (network unreachable, host unreachable, port unreachable, fragmentation needed)
- **Type 5 — Redirect** — маршрутизатор повідомляє хосту про кращий маршрут

**ICMP Flood Attack (Ping Flood):**

- Атакуючий надсилає величезну кількість Echo Request до жертви
- Жертва витрачає ресурси на обробку та генерацію Echo Reply
- Різновид — Smurf Attack: Echo Request відправляється на broadcast-адресу з підробленою IP-адресою жертви, і всі хости мережі відповідають жертві
- Захист: обмеження швидкості ICMP (rate limiting), блокування ICMP на фаєрволі (частково), фільтрація на рівні ISP

## DNS детально

DNS (Domain Name System) — розподілена система перетворення доменних імен в IP-адреси.

### Типи DNS-записів

- **A** — відповідність домену до IPv4-адреси (example.com → 93.184.216.34)
- **AAAA** — відповідність домену до IPv6-адреси (example.com → 2606:2800:220:1:...)
- **MX** — поштовий сервер домену (example.com → mail.example.com, пріоритет 10)
- **CNAME** — псевдонім (alias) для іншого доменного імені (www.example.com → example.com)
- **NS** — авторитетний DNS-сервер для домену (example.com → ns1.example.com)
- **TXT** — текстові записи, використовуються для SPF (перевірка відправника пошти), DKIM (цифровий підпис листів), DMARC (політика обробки пошти)
- **PTR** — зворотний запис (reverse DNS), IP-адреса → доменне ім'я
- **SOA** — Start of Authority, основна інформація про зону (первинний сервер, email адміністратора, серійний номер)

### Атаки на DNS

- **DNS Cache Poisoning** — підміна записів у кеші DNS-сервера хибними даними. Атакуючий надсилає підроблені відповіді, щоб DNS-сервер кешував неправильну IP-адресу для домену. Жертви перенаправляються на шкідливі сайти.
- **DNS Amplification** — різновид DDoS-атаки. Атакуючий надсилає DNS-запити з підробленою IP-адресою жертви (IP spoofing). DNS-сервери відповідають жертві великими відповідями. Коефіцієнт підсилення може досягати 50-70x.
- **DNS Tunneling** — використання DNS-протоколу для прихованої передачі даних. Дані кодуються в DNS-запитах та відповідях (наприклад, у піддоменах). Дозволяє обходити фаєрволи, бо DNS-трафік зазвичай дозволений.

### Захист DNS

- **DNSSEC** (DNS Security Extensions) — додає цифрові підписи до DNS-записів. Кожна DNS-зона підписується криптографічним ключем. Клієнт може перевірити автентичність та цілісність DNS-відповіді. Захищає від DNS Cache Poisoning, але не шифрує трафік.
- **DoH** (DNS over HTTPS) — DNS-запити через зашифрований HTTPS (порт 443)
- **DoT** (DNS over TLS) — DNS-запити через TLS (порт 853)

## DHCP

DHCP (Dynamic Host Configuration Protocol) — протокол автоматичного призначення IP-адрес та мережевих параметрів.

### Процес DORA

1. **Discover** — клієнт надсилає широкомовний запит (broadcast) у мережу для пошуку DHCP-сервера
2. **Offer** — DHCP-сервер пропонує клієнту IP-адресу та параметри мережі (маска, шлюз, DNS)
3. **Request** — клієнт обирає пропозицію та надсилає запит на отримання запропонованої адреси
4. **Acknowledge** — сервер підтверджує призначення адреси та надсилає остаточні параметри. Адреса надається в оренду (lease) на визначений час.

### Атаки на DHCP

- **DHCP Starvation** — атакуючий вичерпує весь пул адрес DHCP-сервера, надсилаючи масові запити з підробленими MAC-адресами. Легітимні клієнти не можуть отримати IP-адресу.
- **DHCP Spoofing (Rogue DHCP)** — атакуючий запускає підроблений DHCP-сервер у мережі. Видає клієнтам хибний шлюз (gateway) або DNS-сервер. Дозволяє перехоплювати трафік (MITM) або перенаправляти на шкідливі ресурси.

### Захист DHCP

- **DHCP Snooping** — функція комутатора, що розрізняє довірені (trusted) та недовірені (untrusted) порти. Тільки довірені порти можуть надсилати DHCP-відповіді (Offer, Acknowledge). Будує таблицю прив'язки IP-MAC-порт для подальшого використання в DAI та IP Source Guard.

## Маршрутизація

### Статична маршрутизація

- Маршрути налаштовуються адміністратором вручну
- Підходить для невеликих мереж з простою топологією
- Не адаптується до змін у мережі автоматично
- Менше навантаження на маршрутизатор (не потрібен протокол маршрутизації)

### Динамічна маршрутизація

Маршрутизатори автоматично обмінюються інформацією про мережі та обирають оптимальні шляхи.

**Протоколи внутрішньої маршрутизації (IGP):**

- **RIP (Routing Information Protocol)** — алгоритм distance vector, метрика — кількість переходів (hop count, максимум 15). Простий, але повільно збігається та не масштабується для великих мереж.
- **OSPF (Open Shortest Path First)** — алгоритм link-state, метрика — вартість (cost, залежить від пропускної здатності). Будує повну карту мережі (link-state database). Швидка збіжність, підтримка VLSM та CIDR. Підходить для великих корпоративних мереж.

**Протокол зовнішньої маршрутизації (EGP):**

- **BGP (Border Gateway Protocol)** — алгоритм path vector, використовується для маршрутизації між автономними системами (AS) в Інтернеті. Обирає маршрут за набором атрибутів (AS-path, next-hop, local preference). Є основою маршрутизації в глобальному Інтернеті.

### BGP Hijacking

- Атакуючий оголошує чужі IP-префікси через BGP як свої
- Трафік, призначений для жертви, перенаправляється через мережу атакуючого
- Може використовуватися для перехоплення, підміни або блокування трафіку
- Захист: RPKI (Resource Public Key Infrastructure) — криптографічна перевірка права оголошення префіксів, фільтрація маршрутів на основі IRR (Internet Routing Registry)

## Wi-Fi безпека

### Стандарти 802.11

- **802.11a** — 5 ГГц, до 54 Мбіт/с
- **802.11b** — 2.4 ГГц, до 11 Мбіт/с
- **802.11g** — 2.4 ГГц, до 54 Мбіт/с
- **802.11n (Wi-Fi 4)** — 2.4/5 ГГц, до 600 Мбіт/с, технологія MIMO
- **802.11ac (Wi-Fi 5)** — 5 ГГц, до 6.9 Гбіт/с, MU-MIMO
- **802.11ax (Wi-Fi 6/6E)** — 2.4/5/6 ГГц, до 9.6 Гбіт/с, OFDMA, покращена робота в щільних середовищах

### Протоколи безпеки Wi-Fi

- **WEP** (Wired Equivalent Privacy) — застарілий та **небезпечний**. Використовує RC4 з 24-бітним IV (вектором ініціалізації). Зламується за хвилини за допомогою інструментів (aircrack-ng). Не використовувати.
- **WPA** (Wi-Fi Protected Access) — перехідний стандарт. Використовує TKIP (Temporal Key Integrity Protocol). Динамічна зміна ключів, але все ще базується на RC4. Вразливий до деяких атак.
- **WPA2** — поточний стандарт безпеки. Використовує **AES-CCMP** для шифрування. Режими: Personal (PSK — Pre-Shared Key) та Enterprise (802.1X + RADIUS).
- **WPA3** — найновіший стандарт. Використовує **SAE** (Simultaneous Authentication of Equals) замість PSK. Захист від офлайн-атак підбору пароля. Forward Secrecy — компрометація пароля не розкриває попередній трафік. Обов'язкове використання PMF (Protected Management Frames).

### Атаки на Wi-Fi

- **Evil Twin** — атакуючий створює підроблену точку доступу з тим самим SSID, що й легітимна. Жертви підключаються до фальшивої мережі, і весь їхній трафік перехоплюється.
- **Deauthentication Attack** — атакуючий надсилає підроблені deauth-фрейми від імені точки доступу. Клієнти відключаються від мережі. Використовується для примусового перепідключення (захоплення handshake для WPA2 PSK).
- **KRACK** (Key Reinstallation Attack) — вразливість у протоколі WPA2 (four-way handshake). Атакуючий примушує клієнта повторно використати одноразовий ключ (nonce). Дозволяє розшифрувати та підмінити трафік. Виправлено оновленнями безпеки.

## Мережеві атаки

### Атаки на канальному рівні

- **ARP-spoofing** — підміна MAC-адреси в ARP-таблиці для Man-in-the-Middle
- **MAC-flooding** — переповнення таблиці MAC-адрес комутатора
- **VLAN hopping** — несанкціонований доступ між VLAN

### Атаки на мережевому рівні

- **IP-spoofing** — підробка IP-адреси відправника
- **Сканування портів** — виявлення відкритих сервісів (Nmap)
- **DoS/DDoS** — відмова в обслуговуванні через перевантаження

### Атаки на прикладному рівні

- **DNS-spoofing** — підміна DNS-відповідей
- **Phishing** — соціальна інженерія через підроблені сайти/листи
- **Man-in-the-Middle (MITM)** — перехоплення та модифікація трафіку

## Засоби мережевої безпеки

### Фаєрволи (міжмережеві екрани)

- **Пакетний фільтр (L3-L4)** — фільтрація за IP, портами, протоколами
- **Stateful фаєрвол** — відстежує стан з'єднань
- **Проксі-фаєрвол (L7)** — аналізує вміст прикладного рівня
- **WAF** — Web Application Firewall, захищає веб-застосунки
- **NGFW** — Next-Generation Firewall, поєднує всі функції + DPI

### IDS / IPS

- **IDS** — Intrusion Detection System — виявляє вторгнення, **сповіщає**
- **IPS** — Intrusion Prevention System — виявляє та **блокує**
- **NIDS/NIPS** — мережевий варіант
- **HIDS/HIPS** — хостовий варіант

Методи виявлення:
- **Сигнатурний** — порівняння з базою відомих атак
- **Аномальний** — виявлення відхилень від нормальної поведінки

### Сегментація мережі

- **VLAN** — логічний поділ мережі на канальному рівні
- **DMZ** — демілітаризована зона для публічних серверів
- **Мікросегментація** — ізоляція окремих робочих навантажень

## Захищені протоколи

### VPN (Virtual Private Network)

Створює зашифрований тунель через незахищену мережу:

- **IPsec** — шифрування на мережевому рівні (AH + ESP)
- **OpenVPN** — працює через TLS, гнучке налаштування
- **WireGuard** — сучасний, швидкий, мінімалістичний
- **L2TP/IPsec** — тунелювання на L2 + шифрування IPsec

### SSL/TLS

- Забезпечує шифрування на транспортному рівні
- **TLS 1.3** — сучасна версія, швидше рукостискання, сильніша криптографія
- Використовується в HTTPS, SMTPS, FTPS

### HTTPS

- HTTP + TLS = захищений веб-трафік
- Використовує сертифікати X.509
- Порт 443 (HTTP — порт 80)

### RADIUS

- Протокол централізованої автентифікації, авторизації та обліку (AAA)
- Використовується для контролю доступу до мережі (Wi-Fi, VPN)
- Працює за моделлю клієнт-сервер
