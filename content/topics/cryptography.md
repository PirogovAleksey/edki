# Криптографічний захист інформації

## Основні поняття

**Криптографія** — наука про методи забезпечення конфіденційності, цілісності та автентичності інформації.

**Криптоаналіз** — наука про методи розкриття шифрів без знання ключа.

**Криптологія** = криптографія + криптоаналіз.

Основні сервіси криптографії:
- **Конфіденційність** — захист від несанкціонованого читання
- **Цілісність** — захист від несанкціонованої зміни
- **Автентичність** — підтвердження особи відправника
- **Неспростовність** — неможливість відмовитися від авторства

## Симетричне шифрування

Використовує **один ключ** для шифрування і дешифрування. Швидке, підходить для великих обсягів даних.

### Блокові шифри

Обробляють дані фіксованими блоками:

- **AES** — стандарт шифрування США, блоки 128 біт, ключі 128/192/256 біт
- **DES** — застарілий стандарт, ключ 56 біт, **не є безпечним**
- **3DES** — потрійне застосування DES, ключ 112/168 біт
- **ДСТУ 7624:2014 (Калина)** — український національний стандарт блокового шифрування, блоки 128/256/512 біт
- **ГОСТ 28147-2009** — стандарт блокового шифрування, блок 64 біт, ключ 256 біт

### Режими шифрування блокових шифрів

Визначені стандартом ISO/IEC 10116:2019:

- **ECB** (Electronic Codebook) — кожен блок шифрується незалежно, **не рекомендується** (не забезпечує дифузію)
- **CBC** (Cipher Block Chaining) — кожен блок XOR-ується з попереднім шифроблоком
- **CTR** (Counter) — використовує лічильник, дозволяє паралельне шифрування
- **CFB** (Cipher Feedback) — блоковий шифр працює як потоковий
- **OFB** (Output Feedback) — генерує гамму, не залежить від шифротексту
- **GCM** (Galois/Counter Mode) — забезпечує автентифіковане шифрування (AEAD)

### Потокові шифри

Шифрують дані побітово/побайтово:

- **RC4** — застарілий, **вважається небезпечним**
- **STRUMOK** — український потоковий шифр
- **ChaCha20** — сучасний потоковий шифр (використовується в TLS 1.3)

## Асиметричне шифрування

Використовує **пару ключів**: публічний (для шифрування) та приватний (для дешифрування).

- **RSA** — базується на складності факторизації великих чисел, ключі 2048-4096 біт
- **Ель-Гамаль (ElGamal)** — базується на складності задачі дискретного логарифмування
- **ECC** — еліптичні криві, менші ключі при тій самій стійкості (256-біт ECC ≈ 3072-біт RSA)

### Протокол обміну ключами Діффі-Хеллмана (DH)

Дозволяє двом сторонам узгодити спільний секретний ключ через відкритий канал. Базується на задачі дискретного логарифмування. Варіант на еліптичних кривих — ECDH.

## Хеш-функції

Перетворюють дані довільної довжини у фіксований хеш (дайджест). Властивості:
- **Односторонність** — неможливо відновити вхідні дані
- **Стійкість до колізій** — важко знайти два різних входи з однаковим хешем
- **Лавинний ефект** — мінімальна зміна входу → значна зміна хешу

Основні алгоритми:

- **MD5** — 128 біт, **вважається небезпечним** (знайдено колізії)
- **SHA-1** — 160 біт, **застарілий**
- **SHA-256** — 256 біт, широко використовується (TLS, блокчейн)
- **SHA-3** — стандарт на основі конструкції Keccak
- **Купина (ДСТУ 7564:2014)** — українська хеш-функція, 256/512 біт

## Електронний цифровий підпис (ЕЦП)

Забезпечує **автентичність** та **цілісність**. Створюється приватним ключем, перевіряється публічним.

Алгоритми:
- **DSA** — стандарт цифрового підпису (FIPS 186)
- **ECDSA** — DSA на еліптичних кривих
- **ДСТУ 4145-2002** — український стандарт ЕЦП на еліптичних кривих

### Інфраструктура відкритих ключів (PKI)

- **Сертифікат X.509** — зв'язує публічний ключ з ідентичністю власника
- **Центр сертифікації (CA)** — видає та підписує сертифікати
- **Список відкликаних сертифікатів (CRL)** — перелік недійсних сертифікатів
- **OCSP** — онлайн-протокол перевірки статусу сертифіката
