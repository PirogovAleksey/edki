# Алгоритмізація та програмування

## 1. Основи алгоритмізації

### 1.1 Поняття алгоритму

**Алгоритм** — це скінченна послідовність чітко визначених дій (кроків), виконання яких призводить до розв'язання поставленої задачі.

Властивості алгоритму:
- **Дискретність** — алгоритм складається з окремих кроків
- **Визначеність** — кожен крок однозначно визначений
- **Скінченність** — алгоритм завершується за скінченну кількість кроків
- **Масовість** — придатність для розв'язання цілого класу задач
- **Результативність** — алгоритм дає результат

### 1.2 Блок-схеми

Блок-схема — графічне зображення алгоритму за допомогою стандартних блоків.

| Блок | Форма | Входи | Виходи | Призначення |
|------|-------|-------|--------|-------------|
| Термінатор | Овал (еліпс) | 0 або 1 | 0 або 1 | Початок / Кінець |
| Процес | Прямокутник | 1 | 1 | Обчислення, присвоєння |
| Умова | Ромб | 1 | 2 | Розгалуження (Так/Ні) |
| Введення/Виведення | Паралелограм | 1 | 1 | Операції вводу/виводу |
| Підпрограма | Прямокутник з подвійними вертикальними лініями | 1 | 1 | Виклик підпрограми |
| З'єднувач | Коло | 1 | 1 | Зв'язок між частинами блок-схеми |

**Важливо:** Блок умови має **1 вхід** і **2 виходи** (Так/Ні).

## 2. Типи даних

### 2.1 Прості типи даних

| Тип | Опис | Приклад |
|-----|------|---------|
| `int` | Ціле число (зазвичай 4 байти) | `int x = 42;` |
| `float` | Дійсне число одинарної точності (4 байти) | `float pi = 3.14f;` |
| `double` | Дійсне число подвійної точності (8 байт) | `double e = 2.718281828;` |
| `char` | Один символ (1 байт) | `char c = 'A';` |
| `bool` | Логічне значення: true або false | `bool flag = true;` |

### 2.2 Складені типи даних

**Масив** — впорядкована сукупність елементів одного типу з доступом за індексом:

```cpp
int arr[5] = {3, 7, 1, 9, 4};  // масив із 5 цілих чисел
arr[0];  // перший елемент = 3 (нумерація з 0)
arr[4];  // останній елемент = 4
```

**Вказівник** — змінна, що зберігає адресу іншої змінної:

```cpp
int x = 10;
int *p = &x;   // p зберігає адресу x
*p = 20;       // розіменування: тепер x = 20
```

## 3. Керуючі конструкції

### 3.1 Умовні оператори

**if / else** — розгалуження за умовою:

```cpp
if (x > 0) {
    // виконується, якщо x додатний
} else if (x == 0) {
    // виконується, якщо x дорівнює нулю
} else {
    // виконується, якщо x від'ємний
}
```

**switch** — множинне розгалуження за значенням:

```cpp
switch (day) {
    case 1: printf("Понеділок"); break;
    case 2: printf("Вівторок"); break;
    case 3: printf("Середа"); break;
    default: printf("Інший день");
}
```

**Важливо:** без `break` виконання «провалюється» у наступний `case`.

### 3.2 Цикли

**for** — цикл із лічильником:

```cpp
// Сума чисел від 1 до n
int sum = 0;
for (int i = 1; i <= n; i++) {
    sum += i;
}
```

**while** — цикл із передумовою (може не виконатися жодного разу):

```cpp
int x = 10;
while (x > 3) {
    x = x - 3;
}
// x = 1 (ітерації: 10 → 7 → 4 → 1)
```

**do-while** — цикл із постумовою (виконується хоча б один раз):

```cpp
int x = 0;
do {
    x++;
} while (x < 5);
// x = 5
```

### 3.3 Типові задачі з циклами (екзаменаційні)

**Визначення значення змінної після циклу:**

```cpp
int k = 0;
for (int i = 0; i < 5; i++) {
    k += i;
}
// k = 0 + 1 + 2 + 3 + 4 = 10
```

```cpp
int s = 1;
for (int i = 1; i <= 4; i++) {
    s = s * i;
}
// s = 1 * 1 * 2 * 3 * 4 = 24 (факторіал 4!)
```

**Пошук максимуму в масиві:**

```cpp
int a[5] = {3, 7, 1, 9, 4};
int max = a[0];
for (int i = 1; i < 5; i++) {
    if (a[i] > max) max = a[i];
}
// max = 9
```

## 4. Функції

### 4.1 Визначення та виклик

Функція — підпрограма, що виконує певну задачу і може повертати результат:

```cpp
int square(int x) {
    return x * x;
}

int result = square(5);  // result = 25
```

### 4.2 Параметри функцій

- **Передача за значенням** — функція отримує копію аргументу, оригінал не змінюється
- **Передача за посиланням** — функція отримує посилання на оригінал і може його змінити
- **Передача за вказівником** — функція отримує адресу змінної

```cpp
void swap(int &a, int &b) {   // передача за посиланням
    int temp = a;
    a = b;
    b = temp;
}
```

### 4.3 Рекурсія

Рекурсія — виклик функцією самої себе. Кожна рекурсивна функція повинна мати **базовий випадок**:

```cpp
int factorial(int n) {
    if (n <= 1) return 1;        // базовий випадок
    return n * factorial(n - 1);  // рекурсивний виклик
}
// factorial(5) = 5 * 4 * 3 * 2 * 1 = 120
```

```cpp
int fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
```

## 5. Вказівники та динамічна пам'ять

### 5.1 Вказівники

```cpp
int x = 10;
int *p = &x;    // p вказує на x
printf("%d", *p);  // виведе 10 (розіменування)
*p = 20;        // x тепер дорівнює 20
```

Операції з вказівниками:
- `&x` — отримати адресу змінної x
- `*p` — розіменування (отримати значення за адресою)
- `p++` — перейти до наступного елемента

### 5.2 Динамічна пам'ять

Виділення та звільнення пам'яті:

```cpp
// C стиль
int *arr = (int*)malloc(n * sizeof(int));
free(arr);

// C++ стиль
int *arr = new int[n];
delete[] arr;
```

**Витік пам'яті (memory leak)** — виникає, коли виділена пам'ять не звільняється. Призводить до поступового вичерпання доступної пам'яті.

## 6. Складність алгоритмів

### 6.1 Нотація Big-O

Складність алгоритму описує залежність часу виконання від розміру вхідних даних n.

| Складність | Назва | Приклад |
|-----------|-------|---------|
| O(1) | Константна | Доступ до елемента масиву за індексом |
| O(log n) | Логарифмічна | Бінарний пошук |
| O(n) | Лінійна | Лінійний пошук, прохід масивом |
| O(n log n) | Лінійно-логарифмічна | Merge sort, quicksort (середній) |
| O(n²) | Квадратична | Bubble sort, insertion sort, selection sort |
| O(2ⁿ) | Експоненціальна | Повний перебір |

### 6.2 Порівняння зростання

Для великих n: O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)

## 7. Основи об'єктно-орієнтованого програмування (ООП)

### 7.1 Класи та об'єкти

**Клас** — шаблон (опис), що визначає структуру та поведінку об'єктів.
**Об'єкт** — екземпляр класу, створений на основі його опису.

```cpp
class Circle {
private:
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() { return 3.14159 * radius * radius; }
};

Circle c(5.0);       // створення об'єкта
double s = c.area(); // виклик методу
```

### 7.2 Чотири принципи ООП

**Інкапсуляція** — приховування внутрішньої реалізації об'єкта та надання доступу через публічний інтерфейс (методи). Захищає дані від некоректного використання.

**Успадкування** — створення нового класу (нащадка) на основі існуючого (батьківського). Нащадок отримує всі властивості й методи батька та може додавати нові або перевизначати існуючі.

```cpp
class Animal {
public:
    virtual void speak() { printf("..."); }
};

class Dog : public Animal {
public:
    void speak() override { printf("Гав!"); }
};
```

**Поліморфізм** — здатність об'єктів різних класів по-різному реагувати на виклик одного й того самого методу. Реалізується через віртуальні функції.

```cpp
Animal *a = new Dog();
a->speak();  // виведе "Гав!" — динамічне зв'язування
```

**Абстракція** — виділення суттєвих характеристик об'єкта та ігнорування несуттєвих. Абстрактний клас — клас, що містить хоча б одну чисту віртуальну функцію і не може бути інстанційований.

```cpp
class Shape {
public:
    virtual double area() = 0;  // чиста віртуальна функція
};
```

### 7.3 Модифікатори доступу

| Модифікатор | Доступ із класу | Доступ із нащадків | Доступ ззовні |
|-------------|:-:|:-:|:-:|
| `public` | Так | Так | Так |
| `protected` | Так | Так | Ні |
| `private` | Так | Ні | Ні |

### 7.4 Конструктори та деструктори

**Конструктор** — спеціальний метод, що викликається при створенні об'єкта:
- Конструктор за замовчуванням — без параметрів
- Конструктор з параметрами
- Конструктор копіювання — створює копію об'єкта

**Деструктор** — спеціальний метод, що викликається при знищенні об'єкта. Ім'я починається з `~`.

```cpp
class MyClass {
public:
    MyClass() { /* конструктор */ }
    MyClass(int x) { /* конструктор з параметром */ }
    MyClass(const MyClass &other) { /* конструктор копіювання */ }
    ~MyClass() { /* деструктор */ }
};
```

**Важливо для екзамену:**
- Конструктор **НЕ може** бути віртуальним (virtual)
- Деструктор **може і повинен** бути віртуальним у базовому класі з віртуальними функціями
- Конструктор може бути inline
- Конструктор не може бути static

### 7.5 Віртуальні функції та vtable

**Віртуальна функція** — функція, оголошена з ключовим словом `virtual`, яка може бути перевизначена в похідному класі.

**Таблиця віртуальних функцій (vtable)** — масив вказівників на віртуальні функції класу. Кожен клас із віртуальними функціями має свою vtable. Кожен об'єкт такого класу містить прихований вказівник (vptr) на vtable свого класу.

Механізм роботи:
1. При виклику віртуальної функції через вказівник на базовий клас
2. Система звертається до vptr об'єкта
3. Знаходить у vtable адресу потрібної функції
4. Викликає реалізацію відповідного похідного класу

Це забезпечує **динамічне (пізнє) зв'язування** — визначення потрібної функції під час виконання програми, а не під час компіляції.

### 7.6 Перевантаження операторів

Перевантаження операторів дозволяє визначити поведінку стандартних операторів (+, -, *, ==, тощо) для користувацьких типів:

```cpp
class Vector2D {
public:
    double x, y;
    Vector2D operator+(const Vector2D &v) const {
        return {x + v.x, y + v.y};
    }
    bool operator==(const Vector2D &v) const {
        return x == v.x && y == v.y;
    }
};
```

## 8. Алгоритми сортування

### 8.1 Бульбашкове сортування (Bubble Sort)

**Принцип:** послідовне порівняння та обмін сусідніх елементів. Більші елементи «спливають» до кінця масиву.

**Складність:** O(n²) — середній та найгірший; O(n) — найкращий (вже відсортований)

```cpp
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}
```

### 8.2 Сортування вставками (Insertion Sort)

**Принцип:** кожен елемент вставляється на правильну позицію у вже відсортовану частину масиву.

**Складність:** O(n²) — середній та найгірший; O(n) — найкращий

```cpp
void insertionSort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

### 8.3 Сортування вибором (Selection Sort)

**Принцип:** на кожному кроці знаходить мінімальний елемент серед невідсортованої частини і ставить його на початок.

**Складність:** O(n²) — завжди (навіть для відсортованого масиву)

```cpp
void selectionSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx])
                minIdx = j;
        }
        int temp = arr[i];
        arr[i] = arr[minIdx];
        arr[minIdx] = temp;
    }
}
```

### 8.4 Швидке сортування (Quicksort)

**Принцип:** вибирається опорний елемент (pivot), масив розбивається на дві частини — менші за pivot і більші за pivot, кожна частина сортується рекурсивно.

**Складність:** O(n log n) — середній; O(n²) — найгірший (невдалий вибір pivot)

```cpp
void quickSort(int arr[], int low, int high) {
    if (low < high) {
        int pivot = arr[high];
        int i = low - 1;
        for (int j = low; j < high; j++) {
            if (arr[j] <= pivot) {
                i++;
                int temp = arr[i];
                arr[i] = arr[j];
                arr[j] = temp;
            }
        }
        int temp = arr[i + 1];
        arr[i + 1] = arr[high];
        arr[high] = temp;
        int pi = i + 1;

        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}
```

### 8.5 Сортування злиттям (Merge Sort)

**Принцип:** масив рекурсивно розбивається навпіл до одиничних елементів, потім відсортовані частини зливаються.

**Складність:** O(n log n) — завжди (найкращий, середній, найгірший)

**Додаткова пам'ять:** O(n)

```cpp
void merge(int arr[], int left, int mid, int right) {
    int n1 = mid - left + 1;
    int n2 = right - mid;
    int L[n1], R[n2];
    for (int i = 0; i < n1; i++) L[i] = arr[left + i];
    for (int j = 0; j < n2; j++) R[j] = arr[mid + 1 + j];
    int i = 0, j = 0, k = left;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) arr[k++] = L[i++];
        else arr[k++] = R[j++];
    }
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];
}

void mergeSort(int arr[], int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;
        mergeSort(arr, left, mid);
        mergeSort(arr, mid + 1, right);
        merge(arr, left, mid, right);
    }
}
```

### 8.6 Порівняння алгоритмів сортування

| Алгоритм | Найкращий | Середній | Найгірший | Пам'ять | Стабільний |
|----------|:---------:|:--------:|:---------:|:-------:|:----------:|
| Бульбашкове | O(n) | O(n²) | O(n²) | O(1) | Так |
| Вставками | O(n) | O(n²) | O(n²) | O(1) | Так |
| Вибором | O(n²) | O(n²) | O(n²) | O(1) | Ні |
| Швидке | O(n log n) | O(n log n) | O(n²) | O(log n) | Ні |
| Злиттям | O(n log n) | O(n log n) | O(n log n) | O(n) | Так |

**Стабільне сортування** — зберігає відносний порядок рівних елементів.

## 9. Алгоритми пошуку

### 9.1 Послідовний (лінійний) пошук

Перевіряє кожен елемент масиву по черзі.

**Складність:** O(n)
**Не вимагає** відсортованого масиву.

```cpp
int linearSearch(int arr[], int n, int key) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == key) return i;
    }
    return -1;  // не знайдено
}
```

### 9.2 Бінарний пошук

Порівнює шуканий елемент із середнім і відкидає половину масиву.

**Складність:** O(log n)
**Вимагає** відсортованого масиву.

```cpp
int binarySearch(int arr[], int n, int key) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = (low + high) / 2;
        if (arr[mid] == key) return mid;
        else if (arr[mid] < key) low = mid + 1;
        else high = mid - 1;
    }
    return -1;  // не знайдено
}
```

### 9.3 Порівняння алгоритмів пошуку

| Алгоритм | Складність | Вимога до масиву |
|----------|:---------:|:----------------:|
| Лінійний пошук | O(n) | Будь-який |
| Бінарний пошук | O(log n) | Відсортований |

Для масиву з 1 000 000 елементів: лінійний пошук — до 1 000 000 порівнянь, бінарний — не більше 20 порівнянь (log₂ 1000000 ≈ 20).

## 10. Бітові оператори

Бітові оператори працюють з окремими бітами числа у двійковому представленні.

| Оператор | Назва | Опис |
|----------|-------|------|
| `<<` | Зсув вліво | Зсуває біти вліво, додаючи нулі справа. Еквівалент множення на 2ⁿ |
| `>>` | Зсув вправо | Зсуває біти вправо, відкидаючи молодші біти. Еквівалент цілочисельного ділення на 2ⁿ |
| `&` | AND (І) | Результат 1, лише якщо обидва біти = 1 |
| `\|` | OR (АБО) | Результат 1, якщо хоча б один біт = 1 |
| `^` | XOR (виключне АБО) | Результат 1, якщо біти різні |
| `~` | NOT (НЕ) | Інвертує всі біти (побітове заперечення) |

### Приклади

```
5 << 2 = 20
  5 у двійковій:  00000101
  зсув вліво на 2: 00010100 = 20

12 & 10 = 8
  12: 1100
  10: 1010
  &:  1000 = 8

6 ^ 3 = 5
  6: 110
  3: 011
  ^: 101 = 5
```

```cpp
int flags = 0;
flags |= (1 << 2);   // встановити біт 2 (flags = 4)
flags |= (1 << 0);   // встановити біт 0 (flags = 5)
bool bit2 = flags & (1 << 2);  // перевірити біт 2 → true
flags &= ~(1 << 2);  // скинути біт 2 (flags = 1)
```

**Практичне застосування:**
- **Прапорці (flags)** — зберігання кількох булевих значень в одному числі
- **Маски** — вибіркове читання або зміна окремих бітів
- **Оптимізація** — швидке множення/ділення на степені двійки через зсуви

## 11. Структури даних

### 11.1 Стек (Stack)

**Стек** — структура даних, що працює за принципом **LIFO** (Last In, First Out — останній увійшов, перший вийшов).

Основні операції:
- **push** — додати елемент на вершину стека
- **pop** — видалити елемент з вершини стека
- **top** (peek) — переглянути верхній елемент без видалення

```cpp
#include <stack>
std::stack<int> s;
s.push(10);    // стек: [10]
s.push(20);    // стек: [10, 20]
s.push(30);    // стек: [10, 20, 30]
int x = s.top();  // x = 30
s.pop();       // стек: [10, 20]
```

Застосування: відкат операцій (undo), обчислення виразів, виклик функцій (стек викликів).

### 11.2 Черга (Queue)

**Черга** — структура даних, що працює за принципом **FIFO** (First In, First Out — перший увійшов, перший вийшов).

Основні операції:
- **enqueue** — додати елемент у кінець черги
- **dequeue** — видалити елемент з початку черги
- **front** — переглянути перший елемент без видалення

```cpp
#include <queue>
std::queue<int> q;
q.push(10);     // черга: [10]
q.push(20);     // черга: [10, 20]
q.push(30);     // черга: [10, 20, 30]
int x = q.front();  // x = 10
q.pop();        // черга: [20, 30]
```

Застосування: обробка завдань у порядку надходження, BFS (пошук у ширину).

### 11.3 Хеш-таблиця (Hash Table)

**Хеш-таблиця** — структура даних, що зберігає пари **ключ → значення** та забезпечує **O(1) середню складність** для операцій вставки, пошуку та видалення.

Принцип роботи:
1. Хеш-функція обчислює індекс (хеш) із ключа
2. Значення зберігається за обчисленим індексом

**Колізія** — ситуація, коли два різних ключі отримують однаковий хеш.

Розв'язання колізій:
- **Метод ланцюжків (chaining)** — кожна комірка містить зв'язаний список елементів із однаковим хешем
- **Відкрита адресація** — при колізії шукається наступна вільна комірка

```cpp
#include <unordered_map>
std::unordered_map<std::string, int> ages;
ages["Олена"] = 20;
ages["Іван"] = 22;
int a = ages["Олена"];  // a = 20
```

### 11.4 Зв'язаний список (Linked List)

**Зв'язаний список** — структура даних, де кожен елемент (вузол) містить дані та вказівник на наступний (і/або попередній) вузол.

**Однозв'язний список** — кожен вузол має вказівник тільки на наступний елемент:

```cpp
struct Node {
    int data;
    Node *next;
};
```

**Двозв'язний список** — кожен вузол має вказівники на наступний і попередній елементи:

```cpp
struct DNode {
    int data;
    DNode *prev;
    DNode *next;
};
```

| Операція | Масив | Зв'язаний список |
|----------|:-----:|:----------------:|
| Доступ за індексом | O(1) | O(n) |
| Вставка на початок | O(n) | O(1) |
| Вставка в кінець | O(1)* | O(n)** |
| Видалення з початку | O(n) | O(1) |

\* — якщо є місце; \*\* — O(1), якщо зберігати вказівник на хвіст.

## 12. Counting Sort (Сортування підрахунком)

**Counting Sort** — алгоритм сортування, який **не використовує порівняння** елементів. Замість цього підраховує кількість входжень кожного значення.

**Складність:** O(n + k), де n — кількість елементів, k — діапазон значень.

**Підходить** для сортування цілих чисел з обмеженим діапазоном значень (наприклад, оцінки від 0 до 100).

**Не підходить** для великого діапазону значень або дійсних чисел.

```cpp
void countingSort(int arr[], int n, int maxVal) {
    int count[maxVal + 1] = {0};

    // Підрахунок кількості кожного значення
    for (int i = 0; i < n; i++) {
        count[arr[i]]++;
    }

    // Відновлення відсортованого масиву
    int idx = 0;
    for (int val = 0; val <= maxVal; val++) {
        while (count[val] > 0) {
            arr[idx++] = val;
            count[val]--;
        }
    }
}
```

**Приклад:** масив [4, 2, 2, 8, 3, 3, 1] → підрахунок: count[1]=1, count[2]=2, count[3]=2, count[4]=1, count[8]=1 → результат: [1, 2, 2, 3, 3, 4, 8].

## 13. Множинне успадкування та Diamond Problem

### 13.1 Множинне успадкування

У C++ клас може наслідувати від **кількох** базових класів одночасно:

```cpp
class A {
public:
    void methodA() { printf("A"); }
};

class B {
public:
    void methodB() { printf("B"); }
};

class C : public A, public B {
    // C має і methodA(), і methodB()
};
```

### 13.2 Diamond Problem (Проблема ромба)

Виникає, коли клас наслідує від двох класів, які мають **спільного предка**:

```
      Base
      /  \
     A    B
      \  /
       C
```

```cpp
class Base {
public:
    int value;
    void show() { printf("Base"); }
};

class A : public Base { };
class B : public Base { };
class C : public A, public B { };
```

**Проблема:** клас C містить **дві копії** Base (через A і через B). Виклик `c.show()` або звернення до `c.value` стає **неоднозначним** — компілятор не знає, яку копію Base використовувати.

### 13.3 Розв'язання: віртуальне успадкування

```cpp
class A : virtual public Base { };
class B : virtual public Base { };
class C : public A, public B { };
```

При **віртуальному успадкуванні** (`virtual`) клас C містить лише **одну копію** Base, що усуває неоднозначність.

## 14. Шаблони (Templates) C++

**Шаблони** — механізм узагальненого програмування, що дозволяє писати код, незалежний від конкретного типу даних.

### 14.1 Шаблони функцій

```cpp
template<typename T>
T maximum(T a, T b) {
    return (a > b) ? a : b;
}

int x = maximum(3, 7);         // T = int, результат 7
double y = maximum(2.5, 1.3);  // T = double, результат 2.5
```

Компілятор автоматично генерує потрібну версію функції для кожного типу, з яким вона викликається (**інстанціювання шаблону**).

### 14.2 Шаблони класів

```cpp
template<typename T>
class Stack {
private:
    T data[100];
    int topIndex;
public:
    Stack() : topIndex(-1) {}
    void push(T value) { data[++topIndex] = value; }
    T pop() { return data[topIndex--]; }
    T top() { return data[topIndex]; }
    bool empty() { return topIndex == -1; }
};

Stack<int> intStack;       // стек цілих чисел
Stack<double> dblStack;    // стек дійсних чисел
Stack<std::string> strStack; // стек рядків
```

**Переваги шаблонів:**
- Повторне використання коду для різних типів
- Типобезпечність (перевірка типів під час компіляції)
- Немає накладних витрат під час виконання (код генерується на етапі компіляції)

## 15. Дружні функції (friend)

**Дружня функція (friend function)** — функція, яка **не є членом класу**, але має доступ до його **private** та **protected** членів.

```cpp
class Rectangle {
private:
    double width, height;
public:
    Rectangle(double w, double h) : width(w), height(h) {}

    // оголошення дружньої функції
    friend double area(const Rectangle &r);
};

// визначення дружньої функції (НЕ член класу)
double area(const Rectangle &r) {
    return r.width * r.height;  // має доступ до private членів
}

Rectangle rect(3.0, 4.0);
double s = area(rect);  // s = 12.0
```

**Ключові особливості:**
- Оголошується всередині класу з ключовим словом `friend`
- Визначається **поза класом** як звичайна функція
- **Не має** вказівника `this` (бо не є членом класу)
- Може бути дружньою для кількох класів одночасно
- Можна також оголосити **дружній клас**: `friend class ClassName;` — всі методи цього класу отримають доступ до private членів

```cpp
class A {
    friend class B;  // всі методи B мають доступ до private членів A
private:
    int secret;
};
```

## 16. Віртуальна пам'ять та сторінковий розподіл

### 16.1 Віртуальна пам'ять

**Віртуальна пам'ять** — механізм, що надає кожному процесу ілюзію власного великого безперервного адресного простору, незалежно від обсягу фізичної пам'яті (RAM).

Кожен процес працює з **віртуальними адресами**, які операційна система транслює у **фізичні адреси** оперативної пам'яті.

### 16.2 Сторінковий розподіл (Paging)

Віртуальний адресний простір розбивається на блоки однакового розміру — **сторінки** (pages), а фізична пам'ять — на **фрейми** (frames) того ж розміру (типово 4 КБ).

**Структура віртуальної адреси:**

```
Віртуальна адреса = [Номер сторінки | Зсув (offset)]
```

- **Номер сторінки** — визначає, яка сторінка потрібна
- **Зсув** — позиція всередині сторінки

### 16.3 Таблиця сторінок (Page Table)

**Таблиця сторінок** — структура даних, що зберігає відповідність між номером віртуальної сторінки та номером фізичного фрейму.

```
Віртуальна сторінка 0 → Фізичний фрейм 5
Віртуальна сторінка 1 → Фізичний фрейм 2
Віртуальна сторінка 2 → Диск (не в RAM)
Віртуальна сторінка 3 → Фізичний фрейм 8
```

Кожен запис у таблиці сторінок містить:
- Номер фізичного фрейму
- Біт присутності (valid/invalid) — чи сторінка завантажена в RAM
- Біти захисту — права доступу (читання, запис, виконання)

### 16.4 Підкачка (Swapping)

**Підкачка** — переміщення сторінок між оперативною пам'яттю (RAM) та диском (swap-файл або swap-розділ).

Коли фізичної пам'яті не вистачає, ОС вивантажує рідко використовувані сторінки на диск, звільняючи місце для нових.

### 16.5 Page Fault (Помилка сторінки)

**Page fault** — переривання, що виникає, коли процес звертається до сторінки, якої **немає у фізичній пам'яті** (біт присутності = 0).

Обробка page fault:
1. Процесор генерує переривання (page fault)
2. ОС знаходить потрібну сторінку на диску
3. Якщо в RAM немає вільних фреймів — вивантажує іншу сторінку на диск
4. Завантажує потрібну сторінку з диска в RAM
5. Оновлює таблицю сторінок
6. Перезапускає інструкцію, що спричинила page fault
